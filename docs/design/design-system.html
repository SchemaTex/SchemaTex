02Branding

The brand is Schematex — a wordmark plus a diagram mark. The mark is a tiny schematic in itself: two square brackets enclose a two-node graph connected by a smooth S-curve. The brackets say "a defined entity / schema"; the curve + endpoints say "a connection between two things." Together: a schema-of-connection — and the curve hides an S for Schematex.

2.1 — The mark
Geometry24×24 base grid
Brackets + S-curve edge. Brackets hug x = 2.5 and x = 21.5, full-height. Two filled endpoint nodes at (16, 7) and (8, 17), joined by a cubic bezier with control points on the midline — a symmetric S.

Why this shape. The bracket is the universal "defined object" notation across every cluster — data structures, set theory, code, BNF grammars. The S-curve edge is the minimum graph that still reads as a graph. Put together, they say schema of a connection — and they hide the S without resorting to a font.

Node radius 2.2u; all strokes 2u. The curve takes --accent; brackets + nodes stay in --text so the mark holds up in monochrome print.

03Icon system

Every diagram type has a dedicated icon in a single consistent visual language. Icons miniaturize the real diagram: a genogram icon shows a genogram, a ladder icon shows two rails + rungs, a circuit icon has a resistor and a component. No generic swooshes. No stand-in pictograms.

3.4 — Do / don't

✓DO

  • Miniaturize the actual diagram — not a metaphor for it.
  • Keep stroke at 1.5 px; let currentColor carry hue.
  • Pair the icon with the diagram's cluster color, not an invented color.
  • Use the icon in nav, tab chips, and doc TOCs — one surface per context.

✗DON'T

  • Substitute emojis (📊 🧬 ⚡) for the icons — they carry brand.
  • Add fills, gradients, or shadows. Outlines only.
  • Invent icons for new diagrams without adding a symbol here first.
  • Shrink below 14 px — the rail structure of ladder / entity breaks.

07Stroke

Three weights. Same discipline as typography. Emphasis is expressed by weight — and for relationships, by doubling the line.

thin1 px
Hairlines, ticks, gridlines
normal2 px
Default body strokes, shape outlines, edges
thick3 px
Node emphasis — proband, star, centerpiece
Line language

Dashed — weak / uncommitted

Ecomap broken link, genogram separated couple, sociogram isolate.

Double — reinforced tie

Marriage, strong affiliation, overdetermined relationship. Node emphasis uses thick stroke, not double.

Dotted — peripheral / latent

Dotted fills at 6–10 % opacity = category background (fishbone rib zones, mindmap branch shading).

08Node anatomy

Every node across every diagram is a composition of the same four layers. Break any one and the family stops feeling coherent.

Four layersanatomy
Each layer pulls from the base 11 tokens — never a handpicked hex. Per-diagram palettes remap only fill.
Default resolutiontoken map
① fill→ fill · fill-muted · per-diagram fill
② stroke→ stroke (2 px normal · 3 px emphasis)
③ text→ text (title) · text-muted (caption)
④ badge→ accent (index) · positive/negative/warn (status)
The badge is optional — used for proband, deceased, active-state. Max one badge per node.

10Repeating motifs

Visual conventions that cross the whole product. Treat them consistently and the surfaces feel coherent without extra work.

Title + description

Every SVG carries <title> + <desc>. Surface them on the page as a caption under the preview.

Broken link

Dashed line always means weak / uncommitted / separated — never "planned" or "future" (which would be dotted).

Slash mark

A small red slash over a node means deceased / eliminated / terminated — uniform across person diagrams and entity trees.

11Do / don't

Hard rules. The authoring library enforces many; these protect the ones it can't.

✓DO

  • Write the brand as Schematex — capital S, lowercase rest.
  • Use var(--accent) as the single CTA color.
  • Use flat fills on anything touching a diagram.
  • Use the 12-slot palette for tags, chips, cluster badges — first 8 only in production.
  • Keep industrial diagrams black & white — dark mode inverts luminance only.
  • Surface <title> + <desc> as caption text under previews.

✗DON'T

  • Write "SchemaTex", "SchemaTeX", or "SCHEMATEX" in prose.
  • Introduce a new accent color or gradient on anything diagram-adjacent.
  • Add drop shadows to diagram frames — 1 px neutral border only.
  • Use slots 8–11 before a new cluster lands.
  • Add font sizes between 9 / 12 / 16 for in-diagram type (14 is probably a label → 12).
  • Recolor industrial diagrams (IEEE 315 / IEC 61131-3 stay B&W).
  • Put emojis in product UI chrome — cluster emoji 👪⚡🏢🐟 are doc-only references.

12Designer checklist

Run this before shipping any new surface.

  • Brand written as Schematex in prose.
  • The surface uses the default base theme tokens verbatim — no custom grays or blues.
  • Dark mode uses Catppuccin Mocha (#1e1e2e, #cdd6f4, #89b4fa).
  • Category chips / charts draw from the first 8 palette slots, not invented colors.
  • Typography stays in 3 tiers for in-diagram content. Marketing scale used only outside frames.
  • Per-diagram palette used only for nodes of that diagram type.
  • Industrial diagrams are not recolored in dark mode — luminance inversion only.
  • No webfont loaded. system-ui + ui-monospace only.
