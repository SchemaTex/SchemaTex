# 10.0 — Ladder Logic Diagram: Implementation Plan

*Standard reference: `docs/system/10-LADDER-LOGIC-STANDARD.md`*
*DiagramType: `"ladder"` | AST: `LadderAST`*

---

## Overview

PLC ladder logic diagrams mimic relay control circuits: left power rail → contacts/coils → right power rail.
Each rung is a horizontal circuit. Parallel branches (OR logic) add vertical rows within a rung.

Layout is entirely **deterministic and rule-based** — no algorithm needed beyond sequential element placement.
Complexity is in function block rendering (timers, counters, math) and parallel branch geometry.

Estimated implementation: **~4–5 hours** for full P0 scope.

## Implementation Steps

### Step 1 — Parser (`parser.ts`) [P0]

DSL format:
```
ladder "Motor Control"

rung 1 "Start/Stop":
  XIC(START_PB)
  XIO(STOP_PB)
  XIC(MOTOR_AUX)
  OTE(MOTOR_CMD)

rung 2 "Running indicator":
  XIC(MOTOR_CMD)
  OTE(PILOT_LIGHT)

rung 3 "Timer":
  XIC(SENSOR_IN)
  TON(TIMER1, PT=5000)
  OTE(OUTPUT1)

rung 4 "Parallel branches (OR)":
  parallel:
    branch:
      XIC(INPUT_A)
    branch:
      XIC(INPUT_B)
      XIC(INPUT_C)
  OTE(OUTPUT2)
```

**Parsing rules:**
- Header: `ladder "title"`
- Rung: `rung N "comment":` followed by indented elements
- Contact: `XIC|XIO|ONS|OSF(TAG [, address: "I:0/0"])`
- Coil: `OTE|OTL|OTU|OTN(TAG [, address: "O:0/0"])`
- Function block: `TON|TOFF|CTU|CTD|ADD|...( TAG, PARAM=value, ... )`
- Parallel block: `parallel:` with `branch:` sub-blocks
- `#` line comments

**Function block parameter parsing:**
```
TON(TIMER1, PT=5000)   → fbType: "TON", tag: "TIMER1", params: { PT: 5000 }
CTU(CTR1, PV=10)       → fbType: "CTU", tag: "CTR1", params: { PV: 10 }
ADD(ADD1, Source_A=TAG1, Source_B=TAG2, Dest=TAG3)
```

**Output:** `LadderAST` with `rungs: LadderRung[]`

**Validation:**
- Coil must be last element (or in coil position) in a rung — warn if not
- Empty rung → parser error
- Unknown element type → parser error with line number

### Step 2 — Rule-Based Layout (`layout.ts`) [P0]

**Constants (from standard §Layout Rules):**
```ts
const LEFT_RAIL_X   = 20;
const RIGHT_RAIL_X_OFFSET = 20;   // from canvas right edge
const RUNG_HEIGHT   = 70;         // between rung baselines
const ELEMENT_WIDTH = 32;         // contact/coil body width
const ELEMENT_HEIGHT = 24;        // contact/coil body height
const FB_WIDTH      = 60;         // function block width
const FB_HEIGHT     = 50;         // function block height
const LABEL_HEIGHT  = 14;         // tag label above element
const RUNG_Y_OFFSET = 40;         // first rung Y from top
const H_SPACING     = 10;         // horizontal gap between elements
const PARALLEL_GAP  = 8;          // vertical gap between parallel branches
const CANVAS_PADDING = 30;
```

**Algorithm:**

```
canvasWidth = LEFT_RAIL_X + RIGHT_RAIL_X_OFFSET + estimated_rung_width

for each rung at index i:
  rungY = RUNG_Y_OFFSET + i * RUNG_HEIGHT

  // Flatten parallel blocks: compute height = sum of branch heights
  // Each parallel adds ELEMENT_HEIGHT per branch + PARALLEL_GAP

  x = LEFT_RAIL_X + H_SPACING    // start of first element

  for each element in rung:
    if element is simple contact/coil:
      node.x = x
      node.y = rungY - ELEMENT_HEIGHT/2
      node.width = ELEMENT_WIDTH
      node.height = ELEMENT_HEIGHT
      x += ELEMENT_WIDTH + H_SPACING

    if element is function block:
      node.x = x
      node.y = rungY - FB_HEIGHT/2
      node.width = FB_WIDTH
      node.height = FB_HEIGHT
      x += FB_WIDTH + H_SPACING

    if element is parallel:
      // Emit vertical wire down at x
      // Each branch: emit elements starting at x + ELEMENT_WIDTH
      // Emit vertical wire back up at end_x
      // Total width = max branch width + 2 * ELEMENT_WIDTH
      parallelLayout(element, x, rungY)
      x += parallel.totalWidth

  // Connect last element wire to right rail
  rightRailX = x + H_SPACING + RIGHT_RAIL_X_OFFSET
```

**LayoutNode fields for ladder:**
```ts
{
  id: "R3_E2",          // "rung3_element2"
  x, y,                 // top-left of element body
  width, height,
  generation: rungIndex,
  individual: element as any,
  // extension:
  elementType: "contact" | "coil" | "function_block" | "branch_start" | "branch_end",
  rungY: number,        // centerline Y
}
```

**LayoutEdge per rung:**
- Horizontal wire segments connecting elements (emit as path)
- Vertical wires for parallel branch connects
- Each stored as `LayoutEdge` with `path` string

**Canvas height:** `RUNG_Y_OFFSET + rungs.length * RUNG_HEIGHT + CANVAS_PADDING`

### Step 3 — Renderer (`renderer.ts`) [P0]

**Power rails:**
```xml
<!-- Left rail -->
<line x1="${LEFT_RAIL_X}" y1="0" x2="${LEFT_RAIL_X}" y2="${height}"
      stroke="#333" stroke-width="4"/>
<!-- Right rail -->
<line x1="${rightRailX}" y1="0" x2="${rightRailX}" y2="${height}"
      stroke="#333" stroke-width="4"/>
```

**Rung numbering:**
```xml
<text x="${LEFT_RAIL_X - 5}" y="${rungY + 4}" text-anchor="end"
      font-size="9" fill="#999">N</text>
```

**Contact symbols (from standard §Symbol Set):**

XIC (Normally Open): Two vertical bars
```xml
<line x1="${x+10}" y1="${y}" x2="${x+10}" y2="${y+h}" stroke="#333" stroke-width="2"/>
<line x1="${x+22}" y1="${y}" x2="${x+22}" y2="${y+h}" stroke="#333" stroke-width="2"/>
<!-- connecting wires: left: x→x+10, right: x+22→x+32 at y+h/2 -->
```

XIO (Normally Closed): XIC + diagonal slash
```xml
<!-- XIC body + -->
<line x1="${x+6}" y1="${y+h}" x2="${x+26}" y2="${y}" stroke="#333" stroke-width="1.5"/>
```

ONS (One-Shot): XIC body + "↑" inside
```xml
<!-- XIC body + -->
<text x="${x+16}" y="${y+h/2+4}" text-anchor="middle" font-size="10">↑</text>
```

**Coil symbols:**

OTE (Output Energize): Parentheses shape
```xml
<path d="M ${x+4},${y} A 12,12 0 0 0 ${x+4},${y+h}
         M ${x+28},${y} A 12,12 0 0 1 ${x+28},${y+h}"
      fill="none" stroke="#333" stroke-width="2"/>
```

OTL (Set): OTE + "S" inside
OTU (Reset): OTE + "R" inside

**Function Block (60×50 rect):**
```xml
<rect x="${x}" y="${y}" width="60" height="50"
      fill="white" stroke="#333" stroke-width="2"/>
<text x="${x+30}" y="${y+14}" text-anchor="middle" font-size="11" font-weight="bold">
  TON
</text>
<!-- Pin labels: IN (top-left), PT (top-right), Q (bottom-left), ET (bottom-right) -->
<text x="${x+4}" y="${y+26}" font-size="8">IN</text>
<text x="${x+4}" y="${y+38}" font-size="8">PT</text>
<!-- pin connection lines -->
<line x1="${x-8}" y1="${y+22}" x2="${x}" y2="${y+22}" stroke="#333" stroke-width="1.5"/>
<line x1="${x-8}" y1="${y+34}" x2="${x}" y2="${y+34}" stroke="#333" stroke-width="1.5"/>
<text x="${x+56}" y="${y+26}" text-anchor="end" font-size="8">Q</text>
<text x="${x+56}" y="${y+38}" text-anchor="end" font-size="8">ET</text>
<line x1="${x+60}" y1="${y+22}" x2="${x+68}" y2="${y+22}" stroke="#333" stroke-width="1.5"/>
<line x1="${x+60}" y1="${y+34}" x2="${x+68}" y2="${y+34}" stroke="#333" stroke-width="1.5"/>
```

**Tag labels (above elements, 9px monospace):**
```xml
<text x="${cx}" y="${y-4}" text-anchor="middle"
      font-size="9" font-family="monospace">START_PB</text>
```

**Address labels (below elements, 8px gray):**
```xml
<text x="${cx}" y="${y+h+10}" text-anchor="middle"
      font-size="8" fill="#666">I:0/1</text>
```

**Rung comment:**
```xml
<text x="${LEFT_RAIL_X + 8}" y="${rungY - 32}"
      font-size="9" fill="#888">-- Start/Stop --</text>
```

**Parallel branches:**
```xml
<!-- Vertical down at branch start -->
<line x1="${bx}" y1="${rungY - branchTotalHeight/2}"
      x2="${bx}" y2="${rungY + branchTotalHeight/2}"
      stroke="#333" stroke-width="1.5"/>
<!-- Vertical down at branch end -->
<line x1="${ex}" y1="${rungY - branchTotalHeight/2}"
      x2="${ex}" y2="${rungY + branchTotalHeight/2}"
      stroke="#333" stroke-width="1.5"/>
<!-- Each branch horizontal wire at its Y level -->
```
