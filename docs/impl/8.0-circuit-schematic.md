# 8.0 — Circuit Schematic: Implementation Plan

*Standard reference: `docs/system/08-CIRCUIT-SCHEMATIC-STANDARD.md`*
*DiagramType: `"circuit"` | AST: `CircuitAST`*

---

## Overview

Circuit schematics use a **positional DSL** (inspired by Schemdraw) where elements are placed one at a time
with a direction, chained from the previous element's end point. Branching is done via `at:` anchor references.

No graph layout solver needed — positions are fully determined by the DSL. The complexity is in
(a) correctly computing anchor coordinates as elements chain together, and
(b) the complete component SVG library.

Estimated implementation: **~5–6 hours** for full P0 scope.

## Implementation Steps

### Step 1 — Parser (`parser.ts`) [P0]

DSL format:
```
circuit "RC Low-Pass Filter"
  V1: voltage_source up [label: "Vs", value: "5V"]
  R1: resistor right [label: "R1", value: "1kΩ"]
    at: V1.start
  C1: capacitor down [label: "C1", value: "100nF"]
  W1: wire down
  W2: wire left [length: 2]
    at: C1.end
  W3: wire up
    at: W2.end
  D1: dot at R1.end
  out: port right [label: "Vout"]
    at: R1.end
```

**Parsing rules:**
- Header: `circuit "title"`
- Component line: `ID: component_type direction [attrs...]`
- `at: ANCHOR_REF` on next line: position this element at that anchor instead of chaining from previous
- `direction`: `right | left | up | down`
- `attrs`: `[key: "value", key: "value"]` — label, value, length, gain, etc.
- **Net declaration**: `net NET_NAME: REF1, REF2, REF3, ...` — connects multiple anchor refs to the same electrical node (auto-generates wires + junction dot)
- `#` comments

**Net syntax example** (multi-terminal node):
```
circuit "Op-Amp Inverter"
  V1: voltage_source up [label: "Vin"]
  R1: resistor right [label: "R1", value: "10kΩ"]
    at: V1.start
  R2: resistor right [label: "Rf", value: "100kΩ"]
    at: R1.end
  U1: opamp down [label: "U1"]
    at: R2.end
  # Net connects R1→R2 junction to op-amp inverting input
  net VMINUS: R1.end, R2.start, U1.minus
  net VOUT: R2.end, U1.out, OUT.start
  OUT: port right [label: "Vout"]
```

**Net handling in parser:**
- `net` lines are parsed after all components; each produces a `CircuitNet` in the AST
- `CircuitNet = { id: string, anchors: string[] }` — list of anchor refs that share the same node
- Parser validates all anchor refs exist; error on undefined

**Supported component types** (from standard §2):
```
resistor, capacitor, inductor, transformer
diode, led, zener, schottky
npn, pnp, nmos, pmos
opamp, voltage_source, current_source, battery
ground, vcc, gnd_signal, gnd_chassis, gnd_digital
wire, dot, label, port
```

**Output:** `CircuitAST` with `components: CircuitComponent[]` and `nets: CircuitNet[]`

```ts
interface CircuitNet {
  id: string;          // net name (e.g., "VOUT")
  anchors: string[];   // anchor refs (e.g., ["R2.end", "U1.out", "OUT.start"])
}
```

### Step 2 — Anchor-Chain Layout (`layout.ts`) [P0]

**Core data structure:** Map of anchor points by `"ID.pinName"` → `{x, y}`

**Algorithm:**
```
cursor = { x: PADDING, y: PADDING }   // current chaining point
anchorMap = new Map<string, Point>()

for each component in ast.components:
  origin = component.at
    ? anchorMap.get(component.at)   // explicit anchor
    : cursor                         // chain from previous

  { start, end, pins } = computePins(component, origin)

  // Register anchors: "ID.start", "ID.end", "ID.center", plus component-specific
  anchorMap.set(`${id}.start`, start)
  anchorMap.set(`${id}.end`, end)
  for each pin in pins: anchorMap.set(`${id}.${pin.name}`, pin.pos)

  cursor = end   // chain continues from end
```

**Component dimensions** (from standard §3):
```ts
const COMPONENT_LENGTH: Record<CircuitComponentType, number> = {
  resistor: 60,
  capacitor: 40,
  inductor: 60,
  transformer: 80,
  diode: 40,
  led: 40,
  npn: 60,
  pnp: 60,
  nmos: 60,
  pmos: 60,
  opamp: 80,
  voltage_source: 50,
  current_source: 50,
  battery: 60,
  wire: 40,     // default; overridden by length attr
  ground: 30,
  vcc: 20,
  dot: 0,
  label: 0,
  port: 20,
};
```

**Direction vector:**
```ts
const DIRECTION_VECTOR: Record<CircuitDirection, [number, number]> = {
  right: [1, 0],
  left: [-1, 0],
  up: [0, -1],
  down: [0, 1],
};
```

**Computing start/end:**
```ts
const [dx, dy] = DIRECTION_VECTOR[component.direction];
const length = component.attrs?.length
  ? parseFloat(component.attrs.length) * BASE_UNIT  // BASE_UNIT = 40px
  : COMPONENT_LENGTH[component.componentType];

start = origin;
end = { x: origin.x + dx * length, y: origin.y + dy * length };
```

**Net wiring (after component placement):**
```
for each net in ast.nets:
  points = net.anchors.map(ref => anchorMap.get(ref))
  // All points should be at same position (user intent: same electrical node)
  // If positions differ, generate Manhattan wire segments connecting them:
  //   - Pick centroid as junction point
  //   - Emit LayoutEdge from each anchor to centroid
  //   - Emit junction dot LayoutNode at centroid (if 3+ anchors)
```

**Canvas sizing:** After all components placed, compute bounding box + PADDING.

**LayoutNode per component:**
```ts
{
  id: component.id,
  x: start.x, y: start.y,
  width: Math.abs(end.x - start.x) + GATE_W,
  height: Math.abs(end.y - start.y) + GATE_H,
  generation: 0,
  individual: component as any,  // carry component data
  // custom extension fields:
  direction: component.direction,
  endX: end.x, endY: end.y,
  pins: pinMap,
}
```

### Step 3 — Component Library (`components.ts`) [P0]

Each component is defined as a function that takes `(startX, startY, direction, length)` and returns SVG element strings.

**Transformation:** All components are authored in "rightward" orientation, then rotated via SVG `transform="rotate(deg, cx, cy)"`:
```ts
const ROTATION: Record<CircuitDirection, number> = {
  right: 0, down: 90, left: 180, up: 270
};
```

**Key component paths (rightward, from standard §Symbol Set):**

**Resistor (zigzag):**
```
M 0,0 L 10,0 L 13,-8 L 18,8 L 23,-8 L 28,8 L 33,-8 L 38,8 L 40,0 L 50,0
```

**Capacitor (parallel plates):**
```
M 0,0 H 17  M 17,-12 V 12  M 23,-12 V 12  M 23,0 H 40
```

**Inductor (arc humps, 4 humps over 60px):**
```
M 0,0 L 5,0 A 5,5 0 0 1 15,0 A 5,5 0 0 1 25,0 A 5,5 0 0 1 35,0 A 5,5 0 0 1 45,0 L 50,0
```

**Diode (triangle + bar):**
```
M 0,0 H 18  M 18,-12 L 32,0 L 18,12 Z  M 32,-12 V 12  M 32,0 H 40
```

**NPN BJT:**
```
// Collector (top): M cx,cy L cx,cy-20
// Base:           M 0,0 H cx-8  M cx-8,cy-14 V cy+14
// Emitter (bot):  M cx,cy+14 L cx+20,cy+26 (arrow outward)
// Body:           M cx-8,cy-14 L cx+8,cy-6  (collector contact)
//                 M cx-8,cy+14 L cx+8,cy+6  (emitter contact)
```

**Op-Amp (triangle):**
```
M 0,15 V 45  M 0,15 L 50,30 L 0,45 Z
// + input pin: (0, 20); - input pin: (0, 40); output: (50, 30)
// Power pins: (25, 0) Vcc; (25, 60) GND
```

**Ground symbols:**
- Earth: `M 0,0 V 15  M -12,15 H 12  M -8,20 H 8  M -4,25 H 4`
- Signal: `M 0,0 V 10  M -8,10 L 0,20 L 8,10 Z` (filled triangle)
- Chassis: `M 0,0 V 10  M -12,10 H 12  M -8,12 V 16  M 0,12 V 16  M 8,12 V 16`

**Transformer (two coil groups):**
```
// Primary (left): 3 humps starting at (0,0) going right to x=30
// Gap:           vertical line at x=30, x=34 (core)
// Secondary:     3 humps from x=34 to x=64
// Polarity dots: <circle r="3"> at primary start, secondary end
```

### Step 4 — Renderer (`renderer.ts`) [P0]

```ts
function renderCircuit(layout: LayoutResult, config: RenderConfig): string {
  // 1. SVG root with role="img" aria-labelledby="lt-circuit-title lt-circuit-desc"
  //    <title id="lt-circuit-title">${ast.title}</title>
  //    <desc id="lt-circuit-desc">Circuit schematic with ${components.length} components</desc>
  //    Each component group: <g data-component-id="${id}" data-component-type="${type}" aria-label="${label || id}">
  // 2. <defs>: arrowheads, LED glow effect (optional)
  // 3. For each LayoutNode:
  //    a. Compute rotation from direction
  //    b. Apply transform="translate(x,y) rotate(deg)"
  //    c. Emit component SVG (call components.ts function)
  //    d. Emit label text (offset perpendicular to direction)
  //    e. Emit value text (below label)
  // 4. Junction dots: <circle r="3" fill="#333"> at Dot components
  // 5. Port markers: hollow circle + label
  // 6. Ground/VCC symbols: always upright (counter-rotate to cancel parent transform)
}
```

**Wire rendering:** Wires are just `<line>` elements from start to end.

**Label positioning:** Perpendicular offset from component center:
- rightward → label above (dy = -12)
- downward → label left (dx = -12)
- etc.

**Value text:** Same position as label but shifted further (dy = -24 for rightward).
