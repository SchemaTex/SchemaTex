# 6.0 — Digital Timing Diagram: Implementation Plan

*Standard reference: `docs/system/06-TIMING-STANDARD.md`*
*DiagramType: `"timing"` | AST: `TimingAST`*

---

## Overview

Digital timing diagrams render waveforms over time — logic signals (0/1/x/z), buses (=), and edge markers.
DSL is WaveDrom-compatible (WaveJSON subset). Layout is trivial row-stacking; the complexity is in per-state SVG path generation.

Estimated implementation: **~3–4 hours** for full P0+P1 scope.

## Implementation Steps

### Step 1 — Parser (`parser.ts`) [P0]

Parse the DSL format:
```
timing "Signal Overview" [hscale: 2]
  CLK:    10101010
  DATA:   x==..==x  "A" "B"
  VALID:  0..1....
  ---
  group "Control" {
    CS_N:  1..0....
    RD_N:  1..0....
  }
```

**Key parsing rules:**
- First line: `timing "title" [hscale: N]`
- Signal line: `NAME: wave_string [data_list]`
- `data_list`: quoted strings after wave, mapped to bus state positions (`=`, `2`–`9`)
- Group: `group "label" { ... }`
- `---` separator: horizontal rule (divider signal)
- `#` comments ignored

**Output:** `TimingAST` with `signals: Array<TimingSignal | TimingGroup>`

**Error handling:**
- Unknown state chars → warn, render as `x`
- Mismatched data count → pad with empty strings

### Step 2 — Layout (`layout.ts`) [P0]

Layout is purely row-based. No graph algorithm needed.

**Constants (from standard):**
```ts
const SIGNAL_NAME_WIDTH = 120;
const SIGNAL_HEIGHT = 28;       // per row
const PERIOD_WIDTH = 40;        // pixels per half-period
const ROW_GAP = 4;
const DIVIDER_HEIGHT = 12;
const GROUP_LABEL_HEIGHT = 18;
```

**Algorithm:**
1. Flatten groups into rows (preserving group label rows)
2. Calculate `maxWaveLength` across all signals
3. `canvasWidth = SIGNAL_NAME_WIDTH + maxWaveLength * PERIOD_WIDTH`
4. Stack rows top-to-bottom, accumulate Y offset
5. Each `LayoutNode` carries: `{ id, x: SIGNAL_NAME_WIDTH, y, width: waveWidth, height: SIGNAL_HEIGHT, signal }`

**LayoutResult:**
- `nodes`: one per signal row + one per group label
- `edges`: empty (timing has no edges)
- `width`, `height`: computed totals

### Step 3 — Waveform State Machine (`waveform.ts`) [P0]

This is the core rendering logic. Given a wave string and dimensions, produce SVG path data for each segment.

**State transition table:**

| Previous → Current | SVG transition |
|---------------------|----------------|
| `0` → `0` | horizontal low line |
| `1` → `1` | horizontal high line |
| `0` → `1` | vertical rise (right-angle) |
| `1` → `0` | vertical fall (right-angle) |
| `*` → `x` | cross-hatch pattern fill + mid-line |
| `*` → `z` | dashed mid-line |
| `*` → `=` | diamond shape open, horizontal mid, close diamond |
| `=` → `=` | horizontal mid |
| `.` (repeat) | extend previous state |
| `p/P` | positive clock pulse (narrow high) |
| `n/N` | negative clock pulse (narrow low) |

**Key path patterns (x=segment start, w=PERIOD_WIDTH, h=SIGNAL_HEIGHT):**
```
Low:  `M x,${h} H ${x+w}`
High: `M x,0 H ${x+w}`
Rise: `M x,${h} L ${x+4},0`   (slight diagonal, not vertical)
Fall: `M x,0 L ${x+4},${h}`
Bus open:  `M x,${h/2} L ${x+6},0`
Bus close: `M ${x+w-6},0 L ${x+w},${h/2} L ${x+w-6},${h}`
```

**`x` (unknown) state:** Use SVG `<pattern>` with cross-hatch:
```xml
<pattern id="lt-xhatch" patternUnits="userSpaceOnUse" width="6" height="6">
  <path d="M 0,6 L 6,0 M -1,1 L 1,-1 M 5,7 L 7,5" stroke="#999" stroke-width="0.5"/>
</pattern>
```

**Node markers:** Scan `node` string, map char positions to x coords, emit `<circle r="3">` markers.

### Step 4 — Renderer (`renderer.ts`) [P0]

Render each row:

1. **Signal name column**: `<text>` right-aligned at `SIGNAL_NAME_WIDTH - 8`
2. **Wave body**: call `waveform.renderWave(signal, x, y, width, height)`
3. **Bus data labels**: centered text between diamond transitions
4. **Tick marks**: vertical lines at each period boundary, short above/below grid
5. **Group brackets**: left brace `{` shape + label for group rows
6. **Annotations**: arrow paths between node markers if `node` strings defined

**SVG structure:**
```xml
<svg class="lt-timing" ...>
  <defs><!-- patterns for x/z --></defs>
  <g class="lt-timing-names"><!-- name column --></g>
  <g class="lt-timing-waves"><!-- per-signal wave rows --></g>
  <g class="lt-timing-grid"><!-- period tick lines --></g>
  <g class="lt-timing-annotations"><!-- arrows --></g>
</svg>
```
