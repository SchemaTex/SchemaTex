# 11.0 — Single-Line Diagram (SLD): Implementation Plan

*Standard reference: `docs/system/11-SINGLE-LINE-STANDARD.md`*
*DiagramType: `"sld"` | AST: `SLDAST`*

---

## Overview

Single-line diagrams (SLD) show power distribution hierarchies: utility/generator → transformers →
buses → breakers → loads. Named "single-line" because 3-phase power is represented by a single line.

Layout is **top-down hierarchical** with voltage level bands (shaded horizontal regions).
Connections flow vertically; bus bars are horizontal thick lines.

Estimated implementation: **~4–5 hours** for full P0 scope.

## Implementation Steps

### Step 1 — Parser (`parser.ts`) [P0]

DSL format:
```
sld "Industrial Plant Distribution"
  UTIL  = utility [voltage: "13.8kV", label: "13.8kV Utility"]
  TX1   = transformer [
    rating: "1000kVA",
    primary: "13.8kV",
    secondary: "480V",
    impedance: "5.75%"
  ]
  BUS1  = bus [voltage: "480V", label: "MCC Bus"]
  CB1   = breaker [rating: "800A", label: "Main CB"]
  CB2   = breaker [rating: "200A"]
  CB3   = breaker [rating: "200A"]
  M1    = motor [rating: "100HP", voltage: "480V", label: "Pump Motor"]
  M2    = motor [rating: "50HP", voltage: "480V", label: "Fan Motor"]
  PNL1  = load [rating: "100A", label: "Panel LP-1"]
  CT1   = ct [ratio: "800:5", label: "CT1"]
  REL1  = relay [device: "51", label: "OC Relay"]

  UTIL  -> TX1
  TX1   -> CB1
  CB1   -> BUS1
  BUS1  -> CB2
  BUS1  -> CB3
  CB2   -> M1
  CB3   -> M2
```

**Parsing rules:**
- Header: `sld "title"`
- Node: `ID = node_type [key: "value", ...]`
- Multi-line attrs: use `[`, then `key: "value"` on each line, then `]`
- Connection: `FROM -> TO [cable: "spec"] [label: "..."]`
- `#` comments

**Supported node types:**
```
utility, generator, transformer, bus,
breaker, switch, fuse,
motor, load, capacitor_bank,
ct, pt, relay,
solar, wind, ups,
ats, vfd
```

- `ats` (Automatic Transfer Switch): dual-source switching, attrs: `[rating: "400A", sources: 2]`
- `vfd` (Variable Frequency Drive): motor speed control, attrs: `[rating: "50HP", voltage: "480V"]`

**Output:** `SLDAST`:
```ts
{
  type: "sld",
  title: "...",
  nodes: [...],
  connections: [...]
}
```

**Validation:**
- Undefined node in connection → parser error
- Bus connected directly to load (missing breaker) → warning (not error; some valid SLDs do this)

### Step 2 — Hierarchical Layout (`layout.ts`) [P0]

**Phase 1: Assign voltage levels**
- Parse `voltage` attr on each node: extract numeric kV value
- Utility/generator → highest voltage level (level 0)
- Transformer secondary → next level
- Bus → same level as its primary input
- Components fed from bus → same level as bus

**Voltage level bands:**
```ts
// Group nodes by voltage level
// Sort levels descending (highest kV at top)
// Assign band Y ranges: each band = BAND_HEIGHT (120px)
const BAND_HEIGHT = 120;
const BAND_COLORS = ["#F5F5F5", "#FAFAFA"];  // alternating light gray
```

**Phase 2: Topological sort**
- Build adjacency from connections
- Kahn's algorithm → layered order
- Nodes in same voltage band → same Y center
- Distribute X positions within each band

**Phase 3: Coordinate assignment**

```ts
const NODE_SPACING_X = 120;     // horizontal spacing between nodes in same band
const BUS_HEIGHT = 6;           // bus bar stroke-width
const SYMBOL_SIZE = 40;         // typical symbol bounding box

// Buses span the full band width
// Other nodes are centered vertically in their band
// Bus bar Y = band_center_y
// Nodes above/below bus connected with vertical wires
```

**X positioning within band:**
- Group nodes by their band (voltage level)
- Sort by connection order (leftmost source feeds leftmost node)
- Space evenly: `x = band_start_x + nodeIndex * NODE_SPACING_X`

**LayoutNode for SLD:**
```ts
{
  id: node.id,
  x, y,                    // center point
  width: SYMBOL_SIZE,
  height: SYMBOL_SIZE,
  generation: voltageLevel, // repurposed: 0 = highest voltage
  individual: node as any,
  // extension:
  voltageLevel: number,
  nodeType: SLDNodeType,
  bandY: number,           // band center Y
}
```

**LayoutEdge for SLD:**
- Utility/generator → transformer: short vertical line (30px)
- Transformer → bus: vertical line connecting transformer secondary to bus bar
- Bus → breaker: short stub down from bus
- Breaker → load/motor: line down from breaker to load symbol

All connections are orthogonal (Manhattan). Simple straight-line L-paths suffice for top-down layout.

### Step 4 — Renderer (`renderer.ts`) [P0]

```ts
function renderSLD(layout: LayoutResult, config: RenderConfig): string {
  // 1. SVG root with role="img" aria-labelledby="lt-sld-title lt-sld-desc"
  //    <title id="lt-sld-title">${ast.title}</title>
  //    <desc id="lt-sld-desc">Single-line diagram with ${nodes.length} equipment items across ${bands.length} voltage levels</desc>
  //    Each equipment: <g data-equipment-id="${id}" data-type="${nodeType}" aria-label="${label || id}: ${rating}">
  // 2. <defs>: arrowhead marker
  // 3. Voltage level bands (shaded background rects):
  //    for each voltageLevel band: <rect class="lt-sld-band"> with light fill
  //    + band label on left margin: "13.8kV" / "480V" etc.
  // 4. <g class="lt-sld-connections"> — LayoutEdge paths
  //    vertical lines with arrowhead at bottom (direction of power flow)
  //    Cable specs: <text> alongside vertical wire, rotated 90°
  // 5. <g class="lt-sld-equipment"> — per LayoutNode:
  //    a. transform="translate(cx, cy)"
  //    b. symbol SVG from symbols.ts
  //    c. Equipment ID label above (11px bold)
  //    d. Rating label below (9px): "800A / 42kA sym"
  //    e. For transformer: nameplate block beside symbol
  // 6. Bus bars: <line stroke-width="6"> spanning band width
  //    (not LayoutNodes, drawn from layout band metadata)
}
```

**Label layout:**
- ID label: `<text y="-${SYMBOL_SIZE/2 + 6}" font-size="11" font-weight="bold">`
- Rating: `<text y="${SYMBOL_SIZE/2 + 14}" font-size="9" fill="#555">`

**Transformer nameplate (beside transformer symbol):**
```xml
<text x="20" y="-8" font-size="8">1000 kVA</text>
<text x="20" y="2" font-size="8">13.8kV / 480V</text>
<text x="20" y="12" font-size="8">Z=5.75%</text>
```

**Power flow arrow:**
```xml
<marker id="lt-sld-arrow" markerWidth="6" markerHeight="5"
        refX="3" refY="2.5" orient="auto">
  <polygon points="0,0 6,2.5 0,5" fill="#555"/>
</marker>
```
