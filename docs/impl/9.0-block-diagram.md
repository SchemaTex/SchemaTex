# 9.0 — Control Systems Block Diagram: Implementation Plan

*Standard reference: `docs/system/09-BLOCK-DIAGRAM-STANDARD.md`*
*DiagramType: `"blockdiagram"` | AST: `BlockAST`*

---

## Overview

Control systems block diagrams follow the Ogata/Franklin convention: signals flow left→right along the
forward path, feedback paths route below the forward path. The key EE-specific element is the
**summing junction** (circle with ±1 signs) which distinguishes this from generic Mermaid flowcharts.

Layout is **semi-fixed**: forward path is auto-arranged left→right, feedback arcs are auto-routed below.
No graph solver needed — topological sort of the signal graph suffices.

Estimated implementation: **~4–5 hours** for full P0 scope.

## Implementation Steps

### Step 1 — Parser (`parser.ts`) [P0]

DSL format:
```
blockdiagram "PID Speed Controller"
  R   = reference [label: "r(t)"]
  E1  = sum(+R, -Ym)    [label: "e(t)"]
  PID = block("K_p + K_i/s + K_d·s") [role: controller]
  MOT = block("1/(Js+b)") [role: plant]
  SPD = signal [label: "y(t)"]
  SPD -> Ym [label: "speed feedback"]
  Ym  = block("K_t") [role: sensor]
  Ym  -> E1
  D   = disturbance [label: "d(t)"]
  E2  = sum(+PID, +D)
  R   -> E1
  E1  -> PID
  PID -> E2
  E2  -> MOT
  MOT -> SPD
```

**Parsing rules:**
- Header: `blockdiagram "title"` (avoids collision with generic "block" keyword)
- Signal/reference: `ID = reference|signal [label: "..."]`
- Summing junction: `ID = sum(+ID, -ID, ...)` — signed inputs
- Block: `ID = block("expr") [role: plant|controller|sensor|actuator|generic]`
- Disturbance: `ID = disturbance [label: "d(t)"]`
- Connection: `FROM -> TO [label: "..."] [discrete: true]`
- `#` comments

**Output:** `BlockAST`:
```ts
{
  type: "blockdiagram",
  title: "PID Speed Controller",
  blocks: [{ id: "PID", label: "K_p + K_i/s + K_d·s", role: "controller" }, ...],
  sums: [{ id: "E1", inputs: ["+R", "-Ym"] }],
  connections: [{ from: "R", to: "E1" }, ...]
}
```

**Validation:**
- Undefined node in connection → parser error
- Sum with no inputs → parser error

### Step 2 — Layout (`layout.ts`) [P0]

**Phase 1: Build signal graph**
- Nodes: blocks + sums + references + signals
- Edges: from `connections`
- Also add implicit edges from sum inputs (from `sums[].inputs`)

**Phase 2: Topological sort (Kahn's algorithm)**
- Identify forward path: sequence from reference node to output signal
- Detect feedback paths: edges that go rightward→leftward (connect output back toward input)

**Phase 3: Forward path layout**
- Assign X positions: `x = nodeIndex * (BLOCK_WIDTH + H_GAP)`
- Assign Y positions: all forward path nodes at `FORWARD_Y = CANVAS_HEIGHT * 0.35`
- Summing junctions: circle radius 20px
- Blocks: rect `width=BLOCK_WIDTH (100px)`, `height=BLOCK_HEIGHT (50px)`

**Constants:**
```ts
const BLOCK_WIDTH = 100;
const BLOCK_HEIGHT = 50;
const H_GAP = 60;           // between forward-path elements
const FORWARD_Y = 120;      // Y of forward path centerline
const FEEDBACK_Y = 240;     // Y of feedback path
const SUM_RADIUS = 20;
const CANVAS_PADDING = 40;
```

**Phase 4: Feedback path routing**
The standard 4-segment L-path for feedback:
```
From output node right side:
  1. Horizontal right: output.x + BLOCK_WIDTH/2 → feedback_right_x
  2. Vertical down: FORWARD_Y → FEEDBACK_Y
  3. Horizontal left: feedback_right_x → sum.x
  4. Vertical up: FEEDBACK_Y → FORWARD_Y (arriving at sum junction bottom)
```
`feedback_right_x` = rightmost node x + BLOCK_WIDTH + H_GAP/2

**Disturbance input routing:**
Arrow from above (FORWARD_Y - 80) down into target sum junction top.

**LayoutNode:**
```ts
// Blocks and sums both become LayoutNodes
// nodeKind stored in individual.properties
// For sums: width=height=SUM_RADIUS*2 (circle)
// For blocks: width=BLOCK_WIDTH, height=BLOCK_HEIGHT
```

**LayoutEdge path:**
- Forward edges: `M x1,y1 H x2` (simple horizontal)
- Feedback edges: 4-segment L-path as above
- Disturbance edges: `M x,y1 V y2` (vertical arrow down)

### Step 3 — Renderer (`renderer.ts`) [P0]

```ts
function renderBlock(layout: LayoutResult, config: RenderConfig): string {
  // 1. SVG root with role="img" aria-labelledby="lt-block-title lt-block-desc"
  //    <title id="lt-block-title">${ast.title}</title>
  //    <desc id="lt-block-desc">Control systems block diagram with ${blocks.length} blocks, ${sums.length} summing junctions</desc>
  //    Each block: <g data-block-id="${id}" data-role="${role}" aria-label="${label || id}">
  // 2. <defs>: arrow markers, label background rect (for signal labels)
  // 3. <g class="lt-block-connections"> — all LayoutEdge paths
  //    - Add arrowhead at target end
  //    - Signal labels: <text> on midpoint of edge with white rect background
  // 4. <g class="lt-block-nodes">:
  //    For each node:
  //    - Block: <rect> with role-based fill, <text> with LaTeX-like expr
  //    - Sum (circle): <circle> + ± signs at 90°/180°/270° pin positions
  //    - Reference: arrow input from left margin
  //    - Signal/output: arrow label on right margin
  //    - Disturbance: arrow from above
  // 5. Line crossing bridges: white gap rectangle at wire intersections
}
```

**Summing junction rendering:**
```xml
<circle cx="${cx}" cy="${cy}" r="20" fill="white" stroke="#333" stroke-width="1.5"/>
<!-- + sign at each positive input pin -->
<text x="${px}" y="${py}" font-size="14" text-anchor="middle">+</text>
<!-- - sign at negative input pin -->
<text x="${px}" y="${py}" font-size="14" text-anchor="middle">−</text>
```

**Role-based block fill:**
```ts
const BLOCK_FILL: Record<BlockRole, string> = {
  plant:       "#FFFFFF",   // white
  controller:  "#E3F2FD",   // light blue
  sensor:      "#F3E5F5",   // light purple
  actuator:    "#E8F5E9",   // light green
  reference:   "#FFFFFF",
  disturbance: "#FFF8E1",   // light yellow
  generic:     "#FFFFFF",
};
```

**Arrow marker:**
```xml
<marker id="lt-bd-arrow" markerWidth="8" markerHeight="6"
        refX="6" refY="3" orient="auto">
  <polygon points="0,0 8,3 0,6" fill="#333"/>
</marker>
```

**Line crossing bridge (white gap):**
When two wires cross perpendicularly, emit a small white rectangle over the background wire:
```xml
<rect x="${x-4}" y="${y-4}" width="8" height="8"
      fill="white" class="lt-block-bridge"/>
```
(Detect crossings: forward path horizontal segments vs feedback path vertical segments.)

## P1 Enhancements

- **Transfer function rendering**: LaTeX-like fractions `K/s` → `K` over `──` over `s`
- **Discrete-time**: dashed lines for z-domain blocks; z-transform notation in block labels
- **Signal bus**: multiple parallel signal lines grouped with label
- **Bode/Nyquist link**: block diagram nodes link to frequency response plots
