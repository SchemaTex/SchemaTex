# 7.0 — Logic Gate Diagram: Implementation Plan

*Standard reference: `docs/system/07-LOGIC-GATE-STANDARD.md`*
*DiagramType: `"logic"` | AST: `LogicGateAST`*

---

## Overview

Logic gate diagrams render combinational and sequential digital circuits as a directed acyclic graph.
Inputs flow left → right through gates to outputs.

Key algorithmic challenge: **DAG layout** — topological sort + layer assignment + column/row positioning.
Wire routing: **Manhattan (orthogonal only)**.

Estimated implementation: **~5–6 hours** for full P0 scope.

## Implementation Steps

### Step 1 — Parser (`parser.ts`) [P0]

DSL format (declarative netlist):
```
logic "Half Adder" [style: ansi]
  input A, B
  output SUM, CARRY

  G1 = XOR(A, B)         -- sum
  G2 = AND(A, B)         -- carry
  SUM   <- G1
  CARRY <- G2
```

**Parsing rules:**
- Header: `logic "title" [style: ansi|iec]`
- `input ID, ID, ...` — input ports
- `output ID, ID, ...` — output ports
- Gate: `ID = GATE_TYPE(inputs...)` where inputs are port ids or gate ids
- `~ID` prefix = active-low (inversion bubble)
- `OUTPUT_ID <- GATE_ID` — maps gate output to named output port
- Sequential gates: `G3 = DFF(D=G1, CLK=CLK, ~R=RESET_N)` — named pin bindings
- `#` line comments

**Output:** `LogicGateAST`:
```ts
{
  type: "logic",
  title: "Half Adder",
  style: "ansi",
  inputs: [{ id: "A", label: "A" }, { id: "B", label: "B" }],
  outputs: [{ id: "SUM", label: "SUM", from: "G1" }, { id: "CARRY", label: "CARRY", from: "G2" }],
  gates: [
    { id: "G1", gateType: "XOR", inputs: ["A", "B"] },
    { id: "G2", gateType: "AND", inputs: ["A", "B"] }
  ]
}
```

**Validation:**
- Undefined inputs → parser error with line number
- Cycles in netlist → parser error ("logic gates must be acyclic")
- Unknown gate type → parser error

### Step 2 — DAG Layout (`layout.ts`) [P0]

**Algorithm: Sugiyama-lite (layer assignment + crossing minimization)**

```
Phase 1: Topological Sort
  - Build adjacency from inputs → gates → outputs
  - Kahn's algorithm (BFS-based): produce sorted order
  - Detect cycles → throw parse error

Phase 2: Layer Assignment
  - Layer 0: all input ports
  - Layer N+1: all output ports
  - Gates: layer = max(layer of all inputs) + 1
  - This is "longest path" layering (ensures proper data flow direction)

Phase 3: Within-Layer Ordering
  - For each layer, order nodes to minimize wire crossings
  - Simple heuristic: sort by median Y of connected nodes in previous layer
  - (Barycenter heuristic — sufficient for typical gate count <20)

Phase 4: Coordinate Assignment
  - Horizontal: layer * LAYER_SPACING (= 140px)
  - Vertical: evenly space within layer, centered on canvas midpoint
  - NODE_SPACING_Y = 80px (gate height=60 + 20px gap)
```

**Constants:**
```ts
const LAYER_SPACING = 160;     // px between layers (wider for 100px gate width)
const NODE_SPACING_Y = 80;     // px between nodes in same layer
const GATE_WIDTH = 100;        // normalized gate body width (matches standard §1.2: 100×60)
const GATE_HEIGHT = 60;        // normalized gate body height
const PORT_MARGIN = 40;        // left/right margin for input/output ports
```

**LayoutNode additions for logic gate:**
Use `LayoutNode.individual` field → cast to carry gate metadata:
```ts
interface LogicLayoutNode extends LayoutNode {
  nodeKind: "input" | "gate" | "output";
  gateType?: LogicGateType;
  style?: LogicGateStyle;
  inputPins: { id: string; x: number; y: number; isActiveLow: boolean }[];
  outputPin: { x: number; y: number; isActiveLow: boolean };
}
```

### Step 3 — Manhattan Wire Router (`router.ts`) [P0]

Each wire connects from `sourceNode.outputPin` → `targetNode.inputPin[k]`.

**Routing strategy:**
1. Horizontal segment from source output to midpoint X
2. Vertical segment to target Y level
3. Horizontal segment to target input pin

```
source.out → (source.out.x + HALF_LAYER_SPACING, source.out.y)
           → (source.out.x + HALF_LAYER_SPACING, target.in.y)
           → target.in
```

**Active-low wires:** Add bubble circle `<circle r="4">` at pin endpoint.

**Shared wire trunks:** When multiple gates share same source output:
- Route vertical trunk at `source.out.x + STUB_LENGTH` (20px)
- Branch horizontally from trunk to each target

**Path string:** Emit as SVG `d` attribute: `M x0,y0 H x1 V y2 H x3`

**LayoutEdge:** Store computed path in `edge.path`.

### Step 4 — Symbol Library (`symbols.ts`) [P0]

Normalized to **100×60px** coordinate space (matching standard `07-LOGIC-GATE-STANDARD.md` §1.2).
All paths defined as SVG `d` strings. Combinational gates are 100×60; sequential (flip-flops) are 60×80.

### Step 5 — Renderer (`renderer.ts`) [P0]

```ts
function renderLogic(layout: LayoutResult, config: RenderConfig): string {
  // 1. SVG root with computed width/height
  // 2. <defs> — arrowhead marker, bubble, clock triangle
  // 3. <g class="lt-logic-wires"> — all LayoutEdge paths
  // 4. <g class="lt-logic-gates"> — per gate: transform to (x,y), draw symbol
  //    - Gate body path
  //    - Gate label (center of body, 11px bold)
  //    - IEC mode: text label inside rect (&, ≥1, =1, etc.)
  //    - Active-low bubbles at pins
  //    - Sequential: clock triangle at CLK pin
  // 5. <g class="lt-logic-ports"> — input/output port labels + dots
  // 6. <g class="lt-logic-labels"> — gate ID labels (G1, G2...) above gate
}
```

**Wire stroke:** `stroke="#333" stroke-width="1.5" fill="none"`
**Gate body:** `stroke="#333" stroke-width="2" fill="white"`
**Port dot:** `<circle r="3" fill="#333"/>`
