---

## Goal

接收 `DiagramAST`，输出 `LayoutResult`：每个 individual 有精确的 (x, y) 坐标，每条关系有 SVG path 数据。

**这不是通用图布局。** 这是一个专门为 genogram 设计的布局算法，遵循 McGoldrick 标准的视觉规则（详见 `docs/system/01-GENOGRAM-STANDARD.md` Section 4）。

---

## Data Structures

### Internal Graph

在 `DiagramAST` 之上构建内部图结构，方便布局计算：

```ts
interface FamilyUnit {
  id: string;
  partners: [string, string];  // [male_id, female_id] 或按年龄排
  relationship: RelationshipType;
  children: string[];  // ordered by birth year
}

interface GenerationLayer {
  index: number;  // 0 = oldest
  nodes: string[];  // individual IDs in left-to-right order
  familyUnits: FamilyUnit[];  // couples in this generation
}

interface LayoutGraph {
  generations: GenerationLayer[];
  individuals: Map<string, Individual>;
  familyUnits: FamilyUnit[];
}
```

## Algorithm (7 Steps)

### Step 1: Build Family Units

从 AST 的 relationships 中提取 family units：
- 每对 couple relationship → 一个 FamilyUnit
- 同一 couple 的 children 归入同一 FamilyUnit
- 一个人可以在多个 FamilyUnit 中出现（多婚姻）

### Step 2: Assign Generation Indices

**BFS-based generation assignment:**

```
1. 找到所有没有 parent 的 individual → generation 0
2. 他们的 children → generation 1
3. Children 的 children → generation 2
4. ...以此类推
```

**Partner 约束：** couple 中的两人必须在同一 generation。如果 BFS 分配了不同的 generation，取较小值（较老的一代），调整另一方。

**Edge case:** 如果一个人在不同 family unit 中被分配了不同 generation（例如跨代婚姻），取最年长的 generation 并标记为"调整"。

### Step 3: Order Nodes Within Generation

在每个 generation 内部排序，目标：**最小化 parent-child 连线交叉**。

**算法：Barycenter Method**

```
for each generation (top to bottom):
  for each node in this generation:
    if node has parents in generation above:
      barycenter = average(parent positions)
    else:
      keep current position
  sort nodes by barycenter
  resolve ties by birth year
```

**强约束：**
- Male left, female right（在 couple 内部）
- Children ordered by birth year

### Step 4: Assign X Positions

**Bottom-up + top-down combination:**

```
Pass 1 (bottom-up): 从最年轻一代开始
  - 每个 individual: width = nodeWidth (40px)
  - 每个 sibling group: space evenly, total width = n * nodeWidth + (n-1) * siblingSpacing
  - 每个 family unit: couple centered above their children
  - 每个 generation: family units spaced by familySpacing

Pass 2 (top-down): 从最年长一代开始
  - 如果 parent couple 的 X 范围与 children 不对齐，调整 children 居中

Pass 3 (conflict resolution):
  - 检查同一 generation 内是否有重叠
  - 如果重叠，向外推开（shift right）
```

### Step 5: Assign Y Positions

简单：`y = generation * generationSpacing`

Default `generationSpacing = 120px`。

### Step 6: Compute Edge Paths

**Couple line:**
```
Male (x1, y) ────── Female (x2, y)
            midpoint
```
SVG: `<line x1=... y1=... x2=... y2=.../>`

**Parent-child connections:**
```
         couple midpoint
              │
              ▼ (drop line, 40px)
    ┌─────────┼──────────┐  (sibship line)
    │         │          │
    ▼         ▼          ▼
  Child1    Child2     Child3
```

SVG path for parent → children:
1. Vertical line from couple midpoint down by `dropDistance`
2. Horizontal sibship line spanning all children
3. Vertical line from sibship line down to each child's top

**Divorce markers:** 两条短斜线在 couple line 中间。
**Separation:** 一条短斜线。

### Step 7: Package Result

组装 `LayoutResult`：
```ts
{
  width: max_x + padding * 2,
  height: max_y + padding * 2,
  nodes: [...],  // 每个 individual 的 LayoutNode
  edges: [...],  // 每条关系的 LayoutEdge (with SVG path)
}
```

## Edge Cases (Test Each One!)

| Case | 处理策略 |
|------|---------|
| Single person, no relationships | 居中渲染，单个 node |
| Childless couple | Couple line only, no drop/sibship |
| Multiple marriages | 每段婚姻水平排列，共享 individual 在中间 |
| 5+ generations | 垂直滚动，保持间距 |
| Person in 2 family units | 同一 node，连接到两条 couple lines |
| Same-sex couple | 按年龄排（年长左），不按 gender |
| Twins | V-shape (identical) 或 bar (fraternal) from sibship line |
| Large sibling group (8+) | 可能需要 wrap 或压缩 spacing |
| Disconnected subgraphs | 多个独立 family tree 并排排列 |
