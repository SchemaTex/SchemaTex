## Goal

实现一个手写 recursive descent parser，将 genogram DSL 文本解析为 `DiagramAST`（定义在 `src/core/types.ts`）。

**不用任何 parser generator**（无 PEG.js，无 chevrotain，无 nearley）。手写控制每一步，保持零依赖。

## Implementation Plan

### Step 1: Tokenizer / Lexer (`src/diagrams/genogram/parser.ts` 内部)

不需要独立 tokenizer 文件。在 parser 内部逐字符/逐行扫描。

```ts
interface ParserState {
  lines: string[];
  currentLine: number;
  currentCol: number;
  indentStack: number[]; // 用于追踪缩进层级
}
```

处理：
- 空行跳过
- `#` 注释行跳过
- 缩进层级计算（空格数）
- 关键词识别：`genogram`、couple operators (`--`, `-x-`, `-/-`, `~`, `==`)

### Step 2: Header Parsing

```ts
function parseHeader(state: ParserState): { title?: string } {
  // 第一行如果是 "genogram" 或 "genogram \"Title\""
  // 返回 title (optional)

```

### Step 3: Individual Parsing

```ts
function parseIndividual(line: string): Individual {
  // "john [male, 1950, deceased, conditions: heart(full) + diabetes(half-left)]"
  // → { id: "john", sex: "male", birthYear: 1950, status: "deceased", conditions: [...] }
}
```

解析 `[...]` 中的 properties：
- 按逗号分割（但注意 conditions 中的 `+` 不能被逗号分割打断）
- 每个 property 匹配：sex | status | year | conditions | key:value

### Step 4: Relationship Parsing

检测行是否包含 couple operator：
```ts
function parseCoupleRelationship(line: string): { from: string; op: string; to: string; toProps?: string } {
  // "john -- mary" → { from: "john", op: "--", to: "mary" }
  // "john -- susan [female, 1960]" → { from: "john", op: "--", to: "susan", toProps: "[female, 1960]" }
}
```

**注意：** 右侧的 partner 可以带 inline properties（第一次出现时定义）。

### Step 5: Indentation-based Child Grouping

最复杂的部分。行缩进增加 = children of the couple above。

```ts
function parseChildren(state: ParserState, parentIndent: number): Individual[] {
  const children: Individual[] = [];
  while (state.currentLine < state.lines.length) {
    const indent = getIndent(state.lines[state.currentLine]);
    if (indent <= parentIndent) break; // 回到父级或更高 → 结束
    children.push(parseIndividual(state.lines[state.currentLine]));
    state.currentLine++;
  }
  return children;
}
```

### Step 6: Assembly

遍历所有行，组装成 `DiagramAST`：
1. 收集所有 individual definitions → `ast.individuals[]`
2. 收集所有 couple relationships → `ast.relationships[]`（type: married/divorced/etc）
3. 收集所有 parent-child relationships → `ast.relationships[]`（type: parent-child）
4. 去重 individual（同一 ID 出现多次时，merge properties）
5. 验证：所有 relationship 中引用的 ID 必须存在

## Error Handling

Parser 错误必须包含：
- **行号和列号**
- **友好的错误信息**（英文）
- **上下文**（显示出错的那行）

```ts
class ParseError extends Error {
  constructor(
    message: string,
    public line: number,
    public column: number,
    public source: string // 出错的行文本
  ) {
    super(`Line ${line}, col ${column}: ${message}\n  → ${source}`);
  }
}
```

常见错误场景：
- 未定义的 ID 引用：`"Unknown individual 'xyz'. Did you mean 'xyz1'?"`
- 无效 property：`"Unknown property 'mail'. Valid: male, female, unknown, deceased, ..."`
- 缩进错误：`"Unexpected indent. Children must be indented under a couple relationship."`
